Replaces a removed course row while AI searches for a replacement.

|----------|-------|

| Width | 100% of course row container |

### 2. "Updated Learning Plan" Indicator

Appears in chat after the AI updates the plan.

| Property | Value |
|----------|-------|

| Container | flex items-center gap-1.5, no background (inline with AI message prose) |
| Typography | 15px (text-sm) semibold for "Updated learning plan" text |

Appears in AI chat messages when suggesting replacement courses.

| Property | Value |
|----------|-------|

| No new component needed | Existing MessageBubble + ReactMarkdown handles this formatting |

Appear in chat panel after plan generation. Follow existing ContextualPills / StructuredChoiceCard pattern.

| Property | Value |
|----------|-------|

## Interaction Contracts

### I-01: Course Removal Flow

1. User hovers over course row -> edit icon fades in (opacity-0 to opacity-100, existing group-hover pattern)
2. User clicks edit icon -> CourseEditMenu dropdown appears
3. User clicks "Remove" -> menu closes -> course row replaced with ShimmerPlaceholder -> programmatic message sent to chat
4. AI responds in chat with replacement suggestion + "Why this recommendation?" reasoning
5. AI emits updated plan via `data-learning-plan` -> `setPlan()` replaces entire plan -> shimmer row replaced with new course (fade-in via existing fadeSlideIn keyframe, 200ms ease-out)

### I-02: Explore Alternatives Flow

1. User clicks "Explore alternatives" in CourseEditMenu -> menu closes
2. Programmatic message sent to chat: "Explore alternatives for [course name] in [milestone name]"
3. AI responds with recommended replacement + "Why this recommendation?" reasoning
4. AI emits updated plan via `data-learning-plan` -> plan auto-updates

### I-03: Broad Refinement via Chat

1. User types freeform refinement in ChatInput (or clicks refinement pill)
2. AI acknowledges change (1 sentence), searches for courses, rebuilds plan
3. Updated plan replaces current plan via `setPlan()` with fadeSlideIn animation
4. AI confirms what changed in 1-2 sentences
5. New refinement pills appear below AI's response

### I-04: Refinement Pills Display

1. After plan generation, AI sends `suggested_pills` via `data-conversation-state`
2. Phase guard in `plan_generated` allows pill updates (modified guard)
3. StructuredChoiceCard renders 4 pills (2 static + 2 AI-generated)
4. User clicks pill -> text sent as chat message -> pills dismissed -> ChatInput appears
5. ChatInput placeholder changes to "Ask about selected courses..." in plan_generated phase

## Animation Contract

| Animation | Keyframe | Duration | Easing | Usage |
|-----------|----------|----------|--------|-------|
| shimmer | opacity 0.5 -> 1 -> 0.5 | 2s | ease-in-out infinite | ShimmerPlaceholder row while AI searches for replacement |
| fadeSlideIn | opacity 0, translateY(8px) -> opacity 1, translateY(0) | 200ms | ease-out | Updated plan appearance, new course row entrance |
| group-hover opacity | opacity-0 -> opacity-100 | default transition | default | Edit icon reveal on course row hover |

All animations respect `prefers-reduced-motion: reduce` (already handled in globals.css).

## State Contract

| State | Visual Indicator | Chat Panel | Plan Panel |
|-------|-----------------|------------|------------|
| plan_generated (idle) | Refinement pills visible | Pills in bottom bar, ChatInput hidden | Full plan displayed, edit icons on hover |
| plan_generated (pills dismissed) | Chat input visible | ChatInput with "Ask about selected courses..." placeholder | Full plan displayed, edit icons on hover |
| plan_generated (refining) | AI streaming response | Streaming message, input disabled | If removal: shimmer placeholder at removed course slot. If broad: plan unchanged until new plan arrives |
| plan_generated (plan updated) | fadeSlideIn on plan banner | AI confirmation message + new refinement pills | Updated plan with fadeSlideIn animation |
