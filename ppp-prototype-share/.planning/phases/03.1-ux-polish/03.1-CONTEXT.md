# Phase 3.1: UX Polish — Context

**Gathered:** 2026-04-01
**Status:** Ready for planning

<domain>
## Phase Boundary

UX refinements to the conversation and plan experience before Phase 4 (Refinement). Five changes: updated sparkle SVGs, loading screen between entry and LIHP, progressive learning plan module that updates as data is gathered, structured multi/single-choice pills in chat, and concise LLM responses. Plan action stubs (... menus, Start button) are visual only — functional behavior is Phase 4.

</domain>

<decisions>
## Implementation Decisions

### 1. Sparkle SVG Update
- **D-01:** Two sparkle SVG variants from `sparkle/` directory in repo root:
  - `_genAI_sparkle_brand_open.svg` — Used in page header (LIHP header, chat panel header). Open 4-point star with radial gradient purple→blue.
  - `_genAI_sparkle_brand_default.svg` — Used in prompt pills, learning plan module header, and general UI. Two-star design (large + small) with same gradient.
- **D-02:** Replace current `SparkleIcon` component (`src/components/shared/sparkle-icon.tsx`) with two variants: `SparkleOpenIcon` (brand_open) and `SparkleIcon` (brand_default). Both accept className prop for sizing.

### 2. Loading Screen (Entry → LIHP)
- **D-03:** Show a skeleton loading screen between the entry screen and LIHP. Minimum display: 200ms, then show LIHP once the first AI streaming token arrives.
- **D-04:** Loading screen layout matches the Figma screenshot: LIHP header at top, skeleton nav pills, 3x3 grid of skeleton course cards (left), chat panel skeleton with shimmer lines (right). Full-page skeleton that mirrors the LIHP structure.
- **D-05:** Transition flow: Entry screen → user sends message → skeleton loading screen → first AI token arrives (after ≥200ms) → LIHP with chat panel and AI response streaming.

### 3. Progressive Learning Plan Module
- **D-06:** A persistent module at the top of the LIHP main content (above Trending section) that shows the learning plan being built progressively as the AI gathers information.
- **D-07:** Four visual states based on `gathered_info` from conversation state:
  1. **Initial** (no data): "Creating your personalized learning plan" — status bar: "Adding your goal..." (animated blue text) · Skills (gray) · Timeline (gray)
  2. **Goal collected** (goal set, no skills/timeline): Status: "Switch roles to {goal}" (black text, completed) · "Adding skills..." (animated blue) · Timeline (gray)
  3. **Skills collected** (goal + skills, no timeline): Status: "{goal}" · "{skills}" (completed) · "Adding duration and timeline..." (animated blue)
  4. **All collected / Plan generated**: Transitions to the full `LearningPlanBanner` with real plan data
- **D-08:** Module has a light purple/lavender background (approximately `#f0edff` / `bg-[#f5f2ff]`). Sparkle icon (brand_default) in header.
- **D-09:** The module is driven by the existing `gathered_info` state in AppShell (`{ goal, background, constraints }`). Goal maps to state 2, skills (extracted from background/conversation) maps to state 3, constraints (timeline) maps to state 4. The AI's structured metadata already provides this data.
- **D-10:** The "animated" status text for the currently-collecting item uses a typewriter/fade effect with the blue accent color (#0056d2 or the sparkle gradient purple).

### 4. Structured Choice Pills
- **D-11:** AI returns structured options instead of plain string pills. New schema:
  ```
  {
    type: "multi" | "single",
    question: string,
    options: string[],
  }
  ```
  This replaces the current `suggested_pills: string[]` in the conversation state metadata.
- **D-12:** Multichoice UI (from screenshot): Card at bottom of chat panel with question header, pagination ("1/3|6" with prev/next arrows), checkbox list of options, "Another {type}" freeform input at bottom, "Skip For Now" link + "Save" button.
- **D-13:** Single-choice UI (from screenshot): Card with question header, numbered radio-style list, "Something else" freeform input, "Skip For Now" link + "Save" button.
- **D-14:** Pagination for multichoice: show ~6 options per page with prev/next navigation arrows and "X/Y|total" indicator.
- **D-15:** When user clicks "Save", selected options are sent as a chat message (e.g., "Data Analyst, Project Manager" for multi, or "3-6 months" for single). The card disappears after save.
- **D-16:** "Skip For Now" — ignore for prototype implementation.

### 5. Concise LLM Responses
- **D-17:** Update system prompt to instruct the AI to keep responses short (2-3 sentences max for conversational turns). No long explanations or lists unless specifically asked. Direct, friendly tone.

### Visual Stubs (non-functional)
- **D-18:** Full plan view shows "..." menu buttons on milestones and course cards — visual only, no dropdown/action. "Start" button on first course — visual only, no action. These become functional in Phase 4.

- Skeleton shimmer animation style for loading screen
- How the "animated blue text" effect works for the currently-collecting item
- Exact lavender background color value for the progressive module
