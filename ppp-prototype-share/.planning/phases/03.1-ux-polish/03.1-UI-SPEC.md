## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline padding |
| sm | 8px | Pill gaps (`gap-2`), compact element spacing |
| md | 16px | Default element spacing, card padding, section gaps |
| lg | 24px | Section padding, module internal padding |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- Pill border-radius: `rounded-full` (9999px) for prompt pills and CTA buttons
- Card border-radius: 12px (`rounded-xl`) for module cards and banners
- Skeleton item border-radius: 8px (`rounded-lg`) for loading skeleton blocks
- Touch target minimum: 36px height for pill buttons, 44px for primary CTAs

Full-page skeleton shown between entry and LIHP. Mirrors LIHP layout structure.

| Property | Value |
|----------|-------|
| Min display | 200ms (enforce via setTimeout before allowing dismiss) |
| Dismiss trigger | First AI streaming token arrives AND min 200ms elapsed |

| Skeleton bg | `#f2f5fa` |
| Border color | `#dae1ed` |

Layout structure (top to bottom):
- LIHP header bar: logo placeholder (left), nav links placeholder (right). Height 56px.
- Navigation pills row: 5 skeleton pills, each 80x32px with 8px gaps, `rounded-full`.
- Main content area (split layout):
  - Left (65%): Progressive module skeleton (rounded-xl, 80px height) at top. Below: 3x3 grid of skeleton course cards. Each card: 200x240px, `rounded-lg`, internal skeleton lines for title (60% width), partner (40% width), meta row (80% width).
  - Right (35%): Chat panel skeleton. Top: 3 shimmer text lines (100%, 80%, 60% width, 16px height, 8px gaps). Bottom: input bar skeleton (full width, 48px height, `rounded-full`).

### 4. ProgressivePlanModule (new)

Persistent module at top of LIHP main content (above Trending). Shows learning plan being built progressively.

| Property | Value |
|----------|-------|
| Background | `#f5f2ff` (lavender) |
| Border | `1px solid #dae1ed` |
| Border radius | 12px (`rounded-xl`) |
| Padding | 20px (`p-5`) |
| Header | SparkleIcon (brand_default, 16x16) + "Creating your personalized learning plan" in `text-sm font-semibold text-[#0f1114]` |

#### Four States

**State 1 — Initial (no gathered data):**
- Status bar: "Adding your goal..." (animated, `text-[#0056d2]`) + dot separator + "Skills" (`text-[#5b6780]`) + dot separator + "Timeline" (`text-[#5b6780]`)
- Animated text uses CSS `@keyframes pulseOpacity` cycling opacity 0.4 to 1.0 over 1.5s ease-in-out infinite

**State 2 — Goal collected (`gathered_info.goal` set):**
- Status: "{goal}" (`text-[#0f1114] font-semibold`, truncated to 40 chars with ellipsis) + dot + "Adding skills..." (animated blue) + dot + "Timeline" (gray)

**State 3 — Skills collected (`gathered_info.background` set):**
- Status: "{goal}" (black, completed) + dot + "{skills summary}" (black, completed) + dot + "Adding duration..." (animated blue)

**State 4 — All collected / Plan generated:**
- Module crossfades (200ms opacity transition) to `LearningPlanBanner` with real plan data
- Transition: `transition-opacity duration-200`

#### Status Bar Layout

- Horizontal flex row with `gap-2`, items center-aligned
- Dot separators: `\u00b7` character in `text-[#5b6780]`
- Each status item: `text-sm` (15px)
- Completed items: `text-[#0f1114]` with subtle checkmark icon (lucide `Check`, 14px, `text-[#0056d2]`) before text
- Active (collecting) items: `text-[#0056d2]` with pulse animation
- Pending items: `text-[#5b6780]`

Card rendered at bottom of chat panel when AI returns `type: "multi"` structured options.

| Property | Value |
|----------|-------|
| Background | `#ffffff` |
| Border | `1px solid #dae1ed` |
| Border radius | 12px (`rounded-xl`) |
| Padding | 20px (`p-5`) |
| Shadow | `shadow-sm` |

Layout (top to bottom):
- **Question header**: `text-sm font-semibold text-[#0f1114]`. Full question text from AI.
- **Pagination indicator** (top right): "{currentPage}/{totalPages}|{totalItems}" in `text-xs text-[#5b6780]`. Left/right arrow buttons (lucide `ChevronLeft`/`ChevronRight`, 16px). Arrows: `text-[#5b6780]` default, `text-[#0056d2]` on hover, disabled state `opacity-30`.
- **Options list**: Vertical stack, `gap-2`, max 6 items per page.
  - Each option: flex row, `gap-3`, `py-2 px-3`, `rounded-lg`, `hover:bg-[#f2f5fa]`, cursor-pointer.
  - Checkbox: 18x18px square, `rounded` (4px), `border-2 border-[#dae1ed]`. Checked: `bg-[#0056d2] border-[#0056d2]` with white checkmark SVG.
  - Label: `text-sm text-[#0f1114]`.

- **Action row**: flex, `justify-between`, `mt-4`.
  - Left: "Skip For Now" — render as `text-sm text-[#5b6780]` link (visually present but non-functional per D-16).
  - Right: "Save" button — `bg-[#0056d2] text-white rounded-full px-5 py-2 text-sm font-semibold hover:bg-[#0048b0]`.

### 6. StructuredChoiceCard (single-choice variant)

Same container as multichoice. Differences:

- **Options list**: Numbered items instead of checkboxes.
  - Number badge: `w-6 h-6 rounded-full bg-[#e3eeff] text-[#0056d2] text-xs font-semibold` centered. Numbers 1, 2, 3...
  - Selected state: badge becomes `bg-[#0056d2] text-white`, row gets `bg-[#f2f5fa]`.
  - Only one selection allowed (radio behavior).
- **No pagination** (single-choice lists are short).
- **Freeform**: "Something else" placeholder text.
- All other properties identical to multichoice variant.

### 7. Visual Stubs (non-functional)

Per D-18, the following appear visually but have no click behavior:

- **Milestone "..." menu**: `text-[#5b6780]` ellipsis icon button (lucide `MoreHorizontal`, 16px), positioned top-right of each milestone header. `cursor-default` (no pointer), no hover state.
- **Course card "..." menu**: Same as milestone, positioned top-right of each course card.
- **"Start" button**: On first course card only. `bg-[#0056d2] text-white rounded-full px-4 py-1.5 text-xs font-semibold`. `cursor-default`, no hover state change.

## Interaction Contracts

### Entry to Loading Transition

1. User submits message (Enter key or click send)
2. Entry screen fades out: `opacity-0 transition-opacity duration-150`
3. LihpLoadingScreen fades in: `opacity-100 transition-opacity duration-150`
4. Timer starts (200ms minimum)
5. When first AI token arrives AND 200ms elapsed: loading screen fades out, LIHP fades in with same duration

### Progressive Module State Transitions

- State changes are driven by `gathered_info` updates from `data-conversation-state` data parts
- Each transition: 200ms crossfade on the status bar content
- No transition between states 1-3 (status text swaps immediately with fade)
- State 3 to State 4 (plan generated): entire module crossfades to LearningPlanBanner

### Structured Pills Interaction

- Card appears when AI message with structured options finishes streaming
- Multichoice: user checks 1+ options, optionally types freeform, clicks Save
- Single-choice: user clicks one numbered option (or types freeform), clicks Save
- On Save: selected values are concatenated with ", " and sent as a user chat message
- Card animates out: `opacity-0 transition-opacity duration-150`, then removed from DOM
- While card is visible, the regular text input is still usable (card does not block it)

### Skeleton Shimmer

- All skeleton blocks use Tailwind `animate-pulse` (opacity 1 to 0.5, 2s duration, infinite)
- Matches the existing `PlanLoadingSkeleton` pattern already in codebase

## Schema Changes

The `ConversationStateData` type and corresponding Zod schema must be updated:

**Before:**
```typescript
suggested_pills: string[]
```

**After:**
```typescript
suggested_pills: string[] | {
  type: "multi" | "single";
  question: string;
  options: string[];
}
```

The union type allows backward compatibility with existing string pill behavior while supporting structured choice cards.
