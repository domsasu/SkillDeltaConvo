# Phase 3.1 Plan 01: Shared Foundations (Icons, Types, Schemas, Prompt, CSS) Summary

Brand sparkle SVGs (two-star default + open 4-point star) with radial gradient, StructuredPillData union type for multi/single-choice pills, 2-3 sentence conciseness directive, pulseOpacity keyframe for progressive module animation.

## What Was Done

### Task 1: Replace sparkle icons with brand SVGs and update all import sites
- Replaced lucide-react Sparkles with two inline SVG components: `SparkleIcon` (brand_default, two-star) and `SparkleOpenIcon` (brand_open, open 4-point star)
- Both use radial gradient from `#6923DE` to `#3286FF` with unique gradient IDs to prevent SVG collision
- Updated header.tsx and chat-side-panel.tsx to use `SparkleOpenIcon` (page header context)
- Kept `SparkleIcon` for pills, banner, and suggestions (brand_default context)
- Removed lucide-react dependency from sparkle-icon.tsx
- **Commit:** `cb1ca32`

### Task 2: Update types, schemas, system prompt, CSS keyframe, and route handler for structured pills and conciseness
- Added `StructuredPillData` type with `type: "multi" | "single"`, `question`, and `options` fields
- Updated `ConversationStateData.suggested_pills` to `string[] | StructuredPillData` union
- Added `structuredPillSchema` Zod validator and updated `conversationStateSchema` with `z.union`
- Updated system prompt: conciseness directive from 2-4 to 2-3 sentences max, added "Structured Options" section for multi/single-choice pill format
- Added `pulseOpacity` CSS keyframe animation (0%-100% opacity 1, 50% opacity 0.4)
- Propagated union type through app-shell, lihp-page, chat-side-panel, chat-panel, contextual-pills
- **Commit:** `e757f98`

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Propagated StructuredPillData union through component prop chain**
- **Found during:** Task 2
- **Issue:** Changing `ConversationStateData.suggested_pills` to a union caused TypeScript errors in app-shell.tsx (state type), lihp-page.tsx, chat-side-panel.tsx, chat-panel.tsx, and contextual-pills.tsx (all typed as `string[]`)
- **Fix:** Updated state type in app-shell and prop types in all intermediate components to accept `string[] | StructuredPillData`. Updated ContextualPills to normalize structured data to string[] via `pills.options` for backward-compatible rendering.
- **Files modified:** app-shell.tsx, lihp-page.tsx, chat-side-panel.tsx, chat-panel.tsx, contextual-pills.tsx
- **Commit:** e757f98
