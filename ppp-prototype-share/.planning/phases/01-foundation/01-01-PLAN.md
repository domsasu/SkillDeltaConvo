    The module must:
    1. Define a Zod schema `envSchema` with three fields:
       - `OPENAI_API_KEY`: z.string().min(1, "OPENAI_API_KEY is required")
       - `OPENAI_MODEL`: z.string().default("gpt-4.1-mini")
       - `COURSERA_GRAPHQL_ENDPOINT`: z.string().url().default("https://www.coursera.org/graphql-gateway")

    2. Export a `config` object by calling `envSchema.parse()` with `process.env` values. This throws with descriptive Zod errors if validation fails (fail-fast at startup).

    3. Export the inferred type: `export type AppConfig = z.infer<typeof envSchema>;`

    Full implementation:
    ```typescript
    import { z } from "zod";

    const envSchema = z.object({
      OPENAI_API_KEY: z.string().min(1, "OPENAI_API_KEY is required"),
      OPENAI_MODEL: z.string().default("gpt-4.1-mini"),
      COURSERA_GRAPHQL_ENDPOINT: z
        .string()
        .url()
        .default("https://www.coursera.org/graphql-gateway"),
    });

    export type AppConfig = z.infer<typeof envSchema>;

    export const config = envSchema.parse({
      OPENAI_API_KEY: process.env.OPENAI_API_KEY,
      OPENAI_MODEL: process.env.OPENAI_MODEL,
      COURSERA_GRAPHQL_ENDPOINT: process.env.COURSERA_GRAPHQL_ENDPOINT,
    });
    ```

    Do NOT use `process.env.OPENAI_API_KEY!` (non-null assertion) anywhere else in the codebase. All other files import `config` from this module.

    After creating, verify the app still builds: `npm run build`.
