**1. Extend `src/lib/prompts/system-prompt.ts`** — add a plan generation section after the existing Response Format section (per D-01, D-02, D-05, D-07, D-14):

Append this section to the existing system prompt string:

```
## Plan Generation

When the user confirms they want their learning plan generated (e.g., "Generate my learning plan", "Yes, create my plan"), follow this process:

### Step 1: Search for Courses
Call the search_courses tool 3-4 times with targeted queries for each milestone area. Examples:
- For a data analyst goal: "python data analysis fundamentals", "SQL data querying", "data visualization tableau", "data analytics capstone project"
- For a career change to PM: "product management fundamentals", "agile project management", "product strategy", "product analytics"

Each search returns up to 8 C+ courses. Use the results to pick the best courses for each milestone.

### Step 2: Design Milestones
Structure milestones based on the learner's goal type:

**Career change / land a new role:**
- Milestone 1: Foundations — include a Professional Certificate or Specialization as the anchor. Badge: "Professional Certificate" or "Core Track".
- Milestone 2: Core Skills — targeted courses for key job skills.
- Milestone 3: Job Readiness — portfolio projects, capstone, interview prep courses.

**Skill improvement / upskilling:**
- One milestone per skill area the learner wants to develop.
- 2-4 courses per milestone.

The AI has flexibility on milestone count (2-5) and naming. Follow the patterns above as guidelines, not rigid rules.

### Step 3: Calculate Duration
- Use the duration field from each course (e.g., "3 months at 10 hours/week") to estimate total hours.
- Divide total hours by the learner's stated hours/week availability (from gathered_info.constraints) to get timeline.
- Show per-milestone estimated weeks and total plan duration (e.g., "3-6 months").
- If learner did not specify hours/week, default to 6-8 hours/week.

### Step 4: Build the Plan
Call the build_learning_plan tool with the assembled plan data. Include:
- title: "{Role} Learning Plan"
- role, skills (3-5 key skills), totalDuration, hoursPerWeek
- milestones array with name, description, skills, badges, estimatedWeeks, and courseIds (from search results)

### Step 5: Confirm in Chat
After calling build_learning_plan, write a brief confirmation message: "I've created your learning plan! It includes [N] milestones with [M] courses to help you [goal]. Click 'View full plan' to see the details."

Do NOT include the JSON metadata block when generating a plan — the tool calls handle state reporting.

### Course Selection Guidelines
- Prefer Professional Certificates and Specializations for foundation milestones (productType: "PROFESSIONAL_CERTIFICATE" or "SPECIALIZATION").
- Include a mix of difficulty levels: BEGINNER for foundations, INTERMEDIATE/ADVANCED for later milestones.
- Pick courses with skills that match the milestone focus area.
- Avoid duplicate courses across milestones.
- Include courses from reputable partners (Google, IBM, Meta, universities).
```
