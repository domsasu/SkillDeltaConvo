1. Create `src/lib/prompts/system-prompt.ts` with `buildSystemPrompt()` function (per D-07, D-08):

   The system prompt must include these sections:
   - **Role:** "You are a learning plan assistant for Coursera Plus learners."
   - **Goal:** "Gather enough context through natural conversation to create a personalized learning plan."
   - **Priority order** (per CONV-02, CONV-03, CONV-04):
     1. Career goal (role, skills, domain, or vague direction)
     2. Background/experience (current role, transferable skills, past experience)
     3. Constraints (target duration, hours per week available)
   - **Conversation guidelines:**
     - Be warm, conversational, and encouraging
     - Ask one focused question at a time
     - If the user gives detailed context upfront, skip unnecessary questions
     - Acknowledge what the user shared before asking the next question
   - **Readiness criteria** (per D-08): Set ready_for_plan to true when you have a clear career goal or learning direction, at least basic background info, and duration/availability info or reasonable defaults to suggest
   - **Response format:** Instruct the AI to end EVERY response with a fenced JSON metadata block:
     ````
     ```json
     { "gathered_info": { "goal": "string or null", "background": "string or null", "constraints": "string or null" }, "ready_for_plan": false, "suggested_pills": ["pill 1", "pill 2", "pill 3"] }
     ```
     ````
   - **Contextual pill guidelines** (per CONV-06):
     - Before goal gathered: career exploration prompts ("I want to switch to data science", "Help me find a new career", "I want to learn AI/ML")
     - After goal, before background: experience prompts ("I have 3 years in marketing", "I'm a complete beginner", "I have a CS degree")
     - After background, before constraints: time prompts ("5 hours per week", "Finish in 6 months", "I can study full-time")
     - When ready_for_plan is true: action prompts ("Generate my learning plan", "I want to add more detail first")

2. Create `src/app/api/chat/route.ts` with POST handler (per D-01, D-02):

   **Approach:** Start with the tool-call approach from the research (Pattern 1 in RESEARCH.md). Define a `report_conversation_state` tool using the AI SDK `tool()` function with the `conversationStateSchema` from schemas.ts as the input schema. The AI will call this tool alongside its text response to report metadata.

   **Implementation:**
   - Import `streamText`, `convertToModelMessages`, `createUIMessageStream`, `createUIMessageStreamResponse`, `tool` from `'ai'`
   - Import `openai` from `'@ai-sdk/openai'`
   - Import `getConfig` from `'@/lib/config'`
   - Import `buildSystemPrompt` from `'@/lib/prompts/system-prompt'`
   - Import `conversationStateSchema` from `'@/lib/prompts/schemas'`
   - Define `report_conversation_state` tool with `conversationStateSchema` as input schema, no execute function (metadata-only)
   - POST handler:
     a. Parse request body `{ messages }`
     b. Call `getConfig()` for OPENAI_MODEL
     c. Convert messages with `convertToModelMessages(messages)`
     d. Create stream with `createUIMessageStream` — in the `execute` callback:
        - Call `streamText` with model `openai(config.OPENAI_MODEL)`, system prompt from `buildSystemPrompt()`, converted messages, and the tool
        - In `onFinish`, extract tool call result for `report_conversation_state` and write it as a data part: `writer.write({ type: 'data-conversation-state', id: 'state-' + Date.now(), data: toolCallArgs })`
        - Merge the streamText result via `writer.merge(result.toUIMessageStream())`
     e. Return `createUIMessageStreamResponse({ stream })`

   **Fallback strategy (per Pitfall 2 and Open Question 1 in RESEARCH.md):** If the tool-call approach has issues at runtime (e.g., model doesn't reliably call the tool alongside text), add a fallback path in `onFinish` that regex-extracts the JSON metadata block from the text response. The regex pattern: `/```json\s*(\{[\s\S]*?\})\s*```/`. Parse with `conversationStateSchema.safeParse()`. If parse succeeds, write the data part. If neither tool call nor inline JSON is found, write a default state with `ready_for_plan: false` and generic pills.

   **Important API notes (from RESEARCH.md anti-patterns):**
   - Do NOT use `toDataStreamResponse()` — that's v4. Use `toUIMessageStreamResponse()` or `createUIMessageStreamResponse()`.
   - Do NOT import from `'ai/react'` — v6 uses `'@ai-sdk/react'` (server code doesn't need this import anyway).
   - The route handler runs in Node.js runtime (not Edge) since Amplify doesn't support Edge streaming (per D-01).

   Add `export const runtime = 'nodejs';` at the top of the route file.
